时间复杂度分析

符号定义

  • O(…)O(\dots)，复杂度上界
  • Ω(…)\Omega(\dots), 复杂度下界
  • Θ(…)\Theta(\dots), 复杂度确界，f(n)f(n) 是 Θ(g(n))\Theta(g(n)) 当且仅当 f(n)f(n) 是 O(g(n))O(g(n)) 且 f(n)f(n) 是 Ω(g(n))\Omega(g(n)) 。

主定理

对于分治算法，可以使用主定理(master theorem) 分析时间复杂度： 对于时间复杂度递推式:

T(n)=a⋅T(nb)+f(n) T(n)=a\cdot T\left(\frac{n}{b}\right)+f(n)

  • Case 1: f(n)=O(nlog⁡ba−ϵ)whereϵ>0f(n)=O\left(n^{\log_b a-\epsilon}\right) \text{where}~\epsilon>0, 那么 T(n)=Θ(nlog⁡ba)T(n)=\Theta\left(n^{\log_b a}\right)
  • Case 2: f(n)=Θ(nlog⁡ba⋅log⁡2kn)wherek≥0f(n)=\Theta\left(n^{\log _b a} \cdot \log_2^k n\right) \text{where}~k\ge 0，那么 T(n)=Θ(nlog⁡balog⁡2k+1n)T(n)=\Theta\left(n^{\log_b a}\log _2^{k+1} n\right)
  • Case 3: f(n)=Ω(nlog⁡ba+ϵ)whereϵ>0f(n)=\Omega\left(n^{\log_b a+\epsilon}\right) \text{where}~\epsilon>0 ，并且 ∃c,n0s.t∀n≥n0,af(nb)≤cf(n)\exists c, n_0~\text{s.t}~\forall n\ge n_0, af\left(\frac{n}{b}\right)\leq cf(n) 那么 T(n)=Θ(f(n))T(n)=\Theta(f(n))

Examples

对于归并排序：

T(n)=2T(n2)+Θ(n) T(n)=2T\left(\frac{n}{2}\right) + \Theta(n)

适用于 Case 2, k=0k=0 ，那么 T(n)=Θ(nlog⁡2n)T(n)=\Theta\left(n\log_2 n\right)

对于二分查找：

T(n)=T(n2)+Θ(1) T(n)=T\left(\frac{n}{2}\right) + \Theta(1)

因为 nlog⁡21=1n^{\log_2 1}=1 ，同样适用于第二种情况，因此 T(n)=Θ(log⁡2n)T(n)=\Theta\left(\log_2 n\right)

大数乘法

假设现在有两个 2n2^n 位的正整数 a,ba,b，需要计算两者相乘的结果。 考虑将 a,ba,b 的前 2n−12^{n-1} 和后 2n−12^{n-1} 位分开，分别得到 a0,a1,b0,b1wherea0+a1⋅2n−1=a,b0+b1⋅2n−1=ba_0,a_1,b_0,b_1 \text{where}~a_0+a_1\cdot 2^{n-1}=a,b_0+b_1\cdot 2^{n-1}=b ，那么可以得到：

a⋅b=a0⋅b0+2n−1⋅(a1⋅b0+a0⋅b1)+2n⋅a1⋅b1 a \cdot b = a_0\cdot b_0+2^{n-1}\cdot (a_1\cdot b_0+a_0\cdot b_1) + 2^n\cdot a_1\cdot b_1

本身需要计算出 a0⋅b0,a1⋅b0,a0⋅b1,a1⋅b1a_0\cdot b_0, a_1\cdot b_0, a_0\cdot b_1, a_1\cdot b_1 共 44 乘法，但是我们更换计算方式：

A=(a0+a1)⋅(b0+b1)B=a1⋅b1C=a0⋅b0D=a1⋅b0+a0⋅b1 \begin{aligned} A&=(a_0+a_1)\cdot(b_0+b_1) \\ B&=a_1\cdot b_1 \\ C&=a_0\cdot b_0 \\ D&=a_1\cdot b_0+a_0\cdot b_1 \end{aligned} 那么显然有 A−B−C=DA-B-C=D ，而我们只需要计算 1(from A)+1(from B)+1(from C)=31 (\text{from A}) +1 (\text{from B})+1 (\text{from C})=3 次乘法，比先前少 11 次，因此总的时间复杂度递推式变为：

T(n)=3⋅T(n2)+Θ(n) T(n)=3\cdot T\left(\frac{n}{2}\right) + \Theta(n)

其中 Θ(n)\Theta(n) 是计算加法所需要的时间复杂度，根据主定理，可以得到：n=nlog⁡23−log⁡232n=n^{\log_2 3 - \log_2 \frac{3}{2}} ，适用于 Case 1, 因此时间复杂度为 Θ(nlog⁡23)\Theta\left(n^{\log _2 3}\right)

PS: 实际上可以使用 快速傅里叶变换 (𝙵𝙵𝚃\texttt{FFT}) 做到 Θ(nlog⁡2n)\Theta\left(n\log_2 n\right) 的时间复杂度。

矩阵乘法

沿用大数乘法的思路，假设有两个 2n⋅2n2^n\cdot 2^n 的矩阵需要计算乘法，将其分别看成两个 2×22\times 2 的矩阵，其中每一个元素都是一个 2n−1⋅2n−12^{n-1}\cdot 2^{n-1} 的矩阵，然后可以发现需要计算 88 次 2n−1⋅2n−12^{n-1}\cdot 2^{n-1} 的矩阵乘法，但是实际上可以使用加法，减少一次乘法（具体的计算就不展开了），因此可以得到：

T(n)=7⋅T(n2)+Θ(n2) T(n)=7\cdot T\left(\frac{n}{2}\right) + \Theta\left(n^2\right) 根据 n2=nlog⁡27−log⁡272n^2=n^{\log_2 7-\log_2 \frac{7}{2}} ，可以得到 T(n)=Θ(nlog⁡27)T(n)=\Theta\left(n^{\log_2 7}\right) 。

平面最近点对

考虑平面上有 nn 个点 (x1,y1),(x2,y2)…(xn,yn)(x_1,y_1),(x_2,y_2)\dots (x_n,y_n) ，现在需要求出

ANS=min⁡1≤i<j≤n(xi−xj)2+(yi−yj)2 \mathrm{ANS}=\min_{1\leq i<j\leq n} \sqrt{(x_i-x_j)^2+(y_i-y_j)^2}

范围是 1≤n≤2×105,∀i∈[1,n]∩ℤ,0≤|xi|,|yi|≤1091\leq n\leq 2\times 10^5, \forall i\in [1,n]\cap \mathbb{Z}, 0\leq |x_i|, |y_i|\leq 10^9 。

首先将所有的点按照 xx 坐标从小到大排序，使用归并，假设现在需要处理编号（排序后的编号）在 [l,r][l,r] 之间的点，取出 xx 的中位数 xmx_m，然后将 [l,mid][l, \text{mid}] 和 [mid+1,r][\text{mid}+1,r] 这两个子区间的答案分别求出，得到他们的较小值 dd ，并且将两个区间的点按照 yy 从小到大排序。

现在可以认为，只有横坐标在 [xm−d,xm+d][x_m-d,x_m+d] 之间的点才可能更新答案。因此将左右两个子区间的满足这个条件的点分别取出，得到两个点的序列 L,RL,R ，这两个序列也是按照 yy 从小到大排序的。

对于 LL 中的节点 (xLi,yLi)\left(x_{L_i},y_{L_i}\right)，考虑可能可以更新答案的 RR 中的点，显然只有 yy 在 [yLi−d,yLi+d]\left[y_{L_i}-d,y_{L_i}+d\right] 的点才能更新答案，将 [xm,xm+d][x_m,x_m+d] 和 [yLi−d,yLi+d]\left[y_{L_i}-d,y_{L_i}+d\right] 这两个区间组成的矩形分成 66 个一样的小矩形，可以发现这些小矩形的对角线长度都是 56d\frac{5}{6}d ，根据 dd 的定义，每个小矩形最多有 11 个点，因此对于每一个 LL 中的点，只有常数个 RR 中的点可能与其配合并更新答案。可以使用双指针法得到 RR 中可行的区间。

因此时间复杂度的递推式为

T(n)=2T(n2)+Θ(n)=Θ(nlog⁡2n) T(n)=2T\left(\frac{n}{2}\right)+\Theta(n)=\Theta(n\log_2 n) 

struct Point {
    LL x, y;
    Point(LL x = 0, LL y = 0) : x(x), y(y) {}
} p[maxn];

inline LL sqr(LL x) { return x * x; }
inline LL dis(const Point &a, const Point &b) { return sqr(a.x - b.x) + sqr(a.y - b.y); }

LL solve(int l, int r) {
    if (l >= r) return INF;
    int mid = (l + r) >> 1;
    LL midx = p[mid].x, mnd = INF;
    LL res = min(solve(l, mid), solve(mid + 1, r));
    vector<Point> lls, rls(1, Point());
    for (int i = l; i <= mid; i++) if (sqr(midx - p[i].x) < res) lls.push_back(p[i]);
    for (int i = mid + 1; i <= r; i++) if (sqr(midx - p[i].x) < res) rls.push_back(p[i]);
    size_t rgl = 1, rgr = 0;
    for (size_t i = 0; i < lls.size(); i++) {
        while (rgr < rls.size() - 1 && (rls[rgr].y <= lls[i].y || sqr(rls[rgr + 1].y - lls[i].y) <= res)) rgr++;
        while (rgl <= rls.size() - 1 && rls[rgl].y < lls[i].y && sqr(rls[rgl].y - lls[i].y) > res) rgl++;
        for (size_t j = rgl; j <= rgr; j++) mnd = min(mnd, dis(lls[i], rls[j]));
    }
    rgl = l, rgr = mid + 1;
    lls.clear();
    while (rgl <= mid && rgr <= r) {
        if (p[rgl].y < p[rgr].y) lls.push_back(p[rgl++]);
        else lls.push_back(p[rgr++]);
    }
    while (rgl <= mid) lls.push_back(p[rgl++]);
    while (rgr <= r) lls.push_back(p[rgr++]);
    for (int i = l; i <= r; i++) p[i] = lls[i - l];
    return min(mnd, res);
